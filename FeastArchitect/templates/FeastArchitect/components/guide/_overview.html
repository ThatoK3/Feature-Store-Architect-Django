The Four Core Components
🗄️

Data Sources

30+ supported databases including PostgreSQL, MongoDB, Kafka, Snowflake, and S3. Each with Debezium CDC support indicators and Spark integration patterns.

FileSource, KafkaSource, PushSource
👤

Entities

Business objects with unique join keys. The backbone of feature relationships enabling point-in-time correct joins across different feature views.

Entity(name="user", join_keys=["user_id"])
📊

Feature Views

Logical groupings of features with TTL, entities, and data sources. Support batch, streaming, and on-demand computation modes.

FeatureView, StreamFeatureView, OnDemandFeatureView
🚀

Feature Services

Model-ready feature bundles with versioning and SLA tracking. Composite services can reference other services for layered abstractions.

FeatureService(name="recommendation_v1")
Quick Start Workflow
1

Design Architecture

Use the canvas to drag data sources, define entities, and connect feature views. The Edge Manager helps validate all connections follow Feast patterns.

2

Configure Ownership

Set team ownership and access processes for each data source. Document security requirements and column-level PII classifications.

3

Generate Code

Export Python files (entities.py, feature_views.py, services.py) or complete JSON for version control. Includes Debezium configs and Spark jobs.

4

Deploy & Monitor

Push to Feast registry, materialize to online store, and monitor via the Data Flow view. Track lineage and performance in real-time.
